## Глобальный обработчик исключений в Spring Boot

**Главная задача кода:** Централизованно отлавливать ошибки (исключения), которые происходят в приложении, не давать программе "упасть" и возвращать пользователю понятный ответ с правильным HTTP-статусом.

### Полный и правильный код (без ошибок)

```java
@RestControllerAdvice
public class GlobalExceptionHandler { 
    private final Logger logger = LoggerFactory.getLogger(GlobalExceptionHandler.class);

    @ExceptionHandler(EntityNotFoundException.class)
    public ResponseEntity<String> handleException(EntityNotFoundException e) {
        String message = e.getMessage();
        logger.warn(message);
        return new ResponseEntity<>(message, HttpStatus.NOT_FOUND);
    }
}

```

---

## 📌 1. Аннотации (Метки для Spring)

* **@RestControllerAdvice** — делает класс глобальным "радаром". Spring понимает, что этот класс должен следить за всеми контроллерами в проекте и перехватывать ошибки, вылетающие из них.
* **@ExceptionHandler** — фильтр ошибок. Указывает, что метод под этой аннотацией должен срабатывать только при возникновении конкретной ошибки (в нашем случае это **EntityNotFoundException.class**).

## 🧩 2. Классы и Объекты

* **GlobalExceptionHandler** — наш собственный класс. Выступает в роли менеджера, который знает, что делать при поломках.
* **EntityNotFoundException** — класс ошибки. Означает, что запрашиваемая сущность (товар, юзер и т.д.) не найдена в базе данных.
* **ResponseEntity** — класс Spring, представляющий собой весь HTTP-ответ целиком ("посылка" для клиента).
* **HttpStatus** — набор стандартных кодов ответа сервера (например, 200 OK, 404 NOT FOUND, 500 ERROR).
* **Logger** — инструмент для записи служебных сообщений в консоль или файл (ведение журнала событий).

## 🛠 3. Методы (Действия)

* **handleException()** — наш метод, внутри которого написана логика спасения ситуации после ошибки.
* **getMessage()** — метод, который достает текстовое сообщение из объекта пойманной ошибки.
* **warn()** — метод логгера, который печатает текст в консоль желтым цветом (уровень "Предупреждение").

---

## ⚙️ 4. Как это работает шаг за шагом (Жизненный цикл ошибки)

1. **Проблема:** Пользователь ищет товар, которого нет. Бизнес-логика выбрасывает ошибку: throw new EntityNotFoundException("Товар не найден").
2. **Перехват:** Аннотация **@RestControllerAdvice** замечает ошибку, а **@ExceptionHandler** проверяет её тип и направляет внутрь метода **handleException**.
3. **Извлечение текста:** Программа берет объект ошибки **e** и вытаскивает из него текст *"Товар не найден"* с помощью метода **e.getMessage()**.
4. **Логирование:** Сервер тихо делает запись в свою консоль: logger.warn(...). Разработчик видит, что был неверный запрос, но сервер продолжает работать.
5. **Ответ клиенту:** Создается новая посылка **ResponseEntity**. В неё кладется текст ошибки и статус **404 (Not Found)**. Эта посылка улетает в браузер пользователя.

## 🔍 5. Построчный разбор сигнатуры метода

Сигнатура — это самая первая строчка объявления метода.

| Часть кода | Что это | Суть человеческим языком |
| --- | --- | --- |
| **public** | Модификатор доступа | Метод открыт, чтобы фреймворк Spring мог его вызвать. |
| **ResponseEntity** | Тип возвращаемого значения | Метод в итоге выдаст HTTP-ответ, внутри которого будет лежать обычный текст (**String**). |
| **handleException** | Имя метода | Просто название. Можно было назвать как угодно. |
| **EntityNotFoundException e** | Входные параметры | Метод принимает пойманную ошибку и кладет её в переменную **e**, чтобы с ней можно было работать внутри. |
